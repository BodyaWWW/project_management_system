## Порівняльна характеристика існуючих засобів вирішення завдання

<span style="color:orange;"><h3> GIT PROJECT </h3></span>  *Це інструмент, що дозволяє створювати дошки в стилі Kanban для впорядкування, пріоритезації та відстеження роботи на рівні репозиторію та організації.Ви npможете створити скільки завгодно проектів у сховищі. Інтерфейс GitHub Projects дозволяє візуалізувати прогрес та завдання проекту, який можна використовувати разом із issues та pull requests, щоб відстежувати роботу, яку необхідно виконати. Система є зручною у використанні та при цьому легко розуміються користувачами.*

 [Посилання на Git Project](https://github.com/features/issues) 
<hr>

<span style="color:orange;"><h3> ASANA </h3></span>  *Проекти над якими працюють розробники можна часто розглядати як одне (або кілька) великих завдань. А щоб було простіше вирішувати велике завдання, його потрібно розділити на дрібніші. Для цього чудово підійде Asana - collaborative task management application. Асана - не просто список завдань, а ціла система дрібних інструментів, які допоможуть замінити майже будь-який додаток з управління завданнями, взаємовідносинами з клієнтами, проектами та багато іншого. Кожна команда може створити для себе зручний робочий простір (workspace). Робочий простір може включати в себе безліч проектів, а кожен проект, в свою чергу, безліч завдань.*

 [Посилання на Asana](https://asana.com/)
 <hr> 

<span style="color:orange;"><h3> TRELLO </h3></span>  *Trello – хмарна програма для управління проектами невеликих груп, розроблена Fog Creek Software.
Trello використовує парадигму для управління проектами, відому як канбан. Trello обмежив підтримку тегів у вигляді десяти кольорових міток, які можна перейменувати. Картки підтримують коментарі, вкладення, терміни виконання та контрольні списки. Trello має API. В даний час підтримуються мобільні платформи додатків iPhone та Android.Проекти зображуються дошками, що містять списки. Списки містять картки, якими зображуються задачі.Картці може бути присвоєно відповідальних за неї користувачів. Користувачі та дошки можуть об'єднуватись в команди.Картки можуть містити коментарі, вкладення, дату завершення та переліки.*

 [Посилання на Trello](https://trello.com/)
 <hr> 

<span style="color:orange;"><h3> PODIO </h3></span>  *Podio стає все більш популярним інструментом для управління проектами, він дозволяє вам організувати будь-який інструмент спілкування для малого бізнесу. Гнучкість також важлива для менеджерів проектів, тому Podio пропонує персоналізовані опції, які дозволяють підприємствам налаштувати його самостійно.Ось дещо з того, що ви можете зробити використовуючи Podio:
1)Спілкуватись в команді
2)Налаштовувати управління завданнями
3)Використовувати систему зберігання файлів
4)Створити індивідуальний стиль* 

 [Посилання на Podio](https://www.podio.com/)
 <hr> 

<span style="color:orange;"><h3> CLARIZEN </h3></span>  *Clarizen - хороший вибір, завдяки відкритим API-інтерфейсам, налаштуванню чату, простоті навігації та комплексній централізації управлінню роботою. Те, за що його вважають кращим - це поєднання управління проектами та управління роботою; програмне рішення, яке створено для малих підприємств які швидко зростають. Веб-додаток для управління проектами зазвичай централізує зв'язок, розклад і завдання, тому кожен розуміє те, що він повинен робити. Інструменти управління роботою виконують і це, а також дозволяють працювати без певних термінів та результатів роботи. Основна міць Clarizen - це простий у використанні інтерфейс, який робить цю амбітну комбінацію управління проектами та робочими можливостями більш корисною навіть для новачків на цій платформі.*

 [Посилання на Clarizen](https://www.clarizen.com/)
 <hr> 

<span style="color:orange;"><h3> TODOIST </h3></span>  *Todoist - це популярне рішення для підвищення продуктивності, яке також містить у собі безліч елементів для командної роботи та ефективного просування проектів. Це також відмінний інструмент для управління проектами для невеликих груп, що робить його гарним вибором для малого бізнесу.Однією з його сильних сторін є призначений для користувача інтерфейс, заснований на системі тегів й схемі кольорів, яка полегшує організацію різних проектів і завдань в рамках проектів. В інтерфейсі також є зручна функція чату, яка робить спілкування швидким та кристально чистим. Що однаково, всім подобається в Todoist  - так це те, наскільки він доступний на багатьох платформах в різних операційних системах. Це робить його надійним вибором для команд, що працюють разом, але які розкидані по всьому світу.*

 [Посилання на TodoIst](https://todoist.com/ru/)
 <hr> 

<span style="color:orange;"><h3> BASECAMP </h3></span>  *Basecamp - це один з найбільш популярних варіантів, це рішення для управління проектами пішло далеко вперед, вже зайнявши пристойний сегмент ринку. Якщо можна було б виразити досвід роботи з Basecamp в декількох словах, то це - зручність використання. Завдяки далекосяжній мобільній інтеграції це стає можливим.Сервіс об'єднує в собі інструменти для комунікації, відстеження завдань, планування і передачі файлів. Робота в Basecamp проходить в єдиному просторі, де розміщені спільні завдання та чат компанії, команди та проекти. Дані компоненти включають 6 елементів для управління проектами: стрічка новин проекту, аркуш завдань, документи та файли, загальний чат, календар та автоматичний чек-ін. Все спілкування відбувається у вигляді блогу, завдання ставляться як to-do списки.*

 [Посилання на BaseCamp](https://basecamp.com/)

## Висновки

*Виконавши аналіз предметної області, можна зазначити, що серед існуючих платформ управління проектами немає таких, які би повністю вирішував проблеми та завдання. Особливу увагу варто звернути на функцію підтримки артефактів проекту, яку належним чином не підтримує жодна з розглянутих систем. Також варто звернутт увагу на офлайн доступ та розробку мобільного додатку для зручного користування на мобільних пристроях. Також варто зробити можливість зв'язуватись з GitHub.*
